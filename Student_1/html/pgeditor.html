Splash Page

Technical Description

The HTML and CSS techniques used in this page are designed to create a visually appealing and functional splash screen that redirects users after a brief delay. The HTML structure is centered around a "div" container that holds the content, including headings ("h1", "h2", "h3"), a paragraph ("p"), and an unordered list ("ul") with list items ("li"). The "meta" tag with http-equiv="refresh" ensures the page automatically redirects to home.html after 5 seconds, enhancing user experience by guiding them to the main content. The CSS styles the page to be fullscreen using height: 100vh and centers the content with display: flex, align-items: center, and justify-content: center. The background image is set to cover the entire screen using background-size: cover and background-position: center, creating an immersive visual effect. The text is bolded with font-weight: bold for emphasis, and the "div" has a semi-transparent dark blue background (rgba(0, 0, 255, 0.6)) with padding and rounded corners (border-radius: 10px) to make the content stand out against the background image. The no-bullets class removes bullet points from the list, ensuring a clean design. Overall, the combination of HTML and CSS creates a cohesive, user-friendly splash screen that effectively communicates the page's purpose while maintaining a professional aesthetic.

Accessibility

In implementing this splash screen, I addressed accessibility by ensuring the content is clear and easy to read for all users. The use of high-contrast colors, such as dark blue background with bold white text, improves readability for users with visual impairments. The font-weight: bold property ensures the text stands out, while the background-size: cover and background-position: center properties ensure the background image does not interfere with text visibility. Additionally, the use of semantic HTML tags like "h1", "h2", and "p" helps screen readers interpret the content structure, making it easier for users with disabilities to navigate. The auto-redirect feature, implemented with the "meta" tag, is complemented by a visible "Loading... Please wait" message, ensuring users are aware of the page's behavior. These design choices collectively enhance accessibility, making the page inclusive for a wider audience.

Volunteer Page

Technical Description

The HTML and CSS techniques used in this page are designed to create an interactive and visually appealing volunteer opportunities page. The HTML structure is organized using a "div" container to hold the content, with sections for volunteer programs, user reviews, and feedback. Each volunteer program is displayed in a card format using "div class="card"", which includes an image ("img"), a heading ("h3"), a paragraph ("p"), and a button ("button"). The scrollable-section class uses display: flex and overflow-x: auto to create a horizontally scrollable section, making it easy for users to browse through multiple programs. The CSS ensures a clean layout with padding, margin, and border-radius for rounded corners, while the background-color properties provide a consistent color scheme. The button class includes a hover effect (:hover) to enhance interactivity. The reviews section uses review-card to display user feedback in a structured format, and the feedback form includes a textarea for user input and emoji ratings for quick feedback. The footer ("div class="footer") provides a link to the student editor page, styled with a contrasting background color for visibility. Overall, the combination of semantic HTML tags and CSS properties ensures a user-friendly, responsive, and visually cohesive design.

Accessibility

In implementing this volunteer opportunities page, I addressed accessibility by ensuring the content is clear, navigable, and usable for all users. I used semantic HTML tags like "h2", "h3", and "p" to structure the content, making it easier for screen readers to interpret and navigate. The images include descriptive alt attributes, providing context for users who rely on screen readers or have images disabled. The scrollable section for volunteer programs is designed with overflow-x: auto, allowing users to scroll horizontally if they cannot use a mouse, and the buttons are large and clickable, ensuring they are easy to interact with. The feedback form includes a textarea for user input and emoji ratings, which are visually distinct and labeled for clarity. Additionally, the color scheme uses high-contrast colors, such as white text on a dark blue background in the footer, to improve readability for users with visual impairments. These design choices ensure the page is inclusive and accessible to a wide range of users.

Content Page

Technical Description

The HTML and CSS techniques used in this page are designed to create a structured, informative, and visually appealing layout for marine conservation content. The HTML uses semantic tags like "header", "section", and "footer" to organize the content logically, making it easier for screen readers and search engines to understand. The "nav" element with a sidebar class provides a fixed navigation menu on the left, allowing users to quickly jump to different sections using anchor links ("a href="#section-id""). The CSS ensures a clean and responsive design, with the sidebar styled to be fixed and full-height, using position: fixed and height: 100%, while the main content ("div class="content"") is offset with margin-left to avoid overlap. Images are made responsive with max-width: 100% and height: auto, ensuring they scale properly on different devices. The box-shadow and border-radius properties add subtle visual enhancements, making sections and images stand out. The top-button class creates a fixed "Go to Top" button at the bottom-right corner, improving navigation for long pages. Overall, the combination of semantic HTML and thoughtful CSS ensures a user-friendly, accessible, and visually cohesive design that effectively communicates marine conservation efforts.

Accessibility

In implementing this marine conservation page, I addressed accessibility by ensuring the content is clear, navigable, and usable for all users. I used semantic HTML tags like "header", "section", and "footer" to structure the content, making it easier for screen readers to interpret and navigate. The sidebar navigation includes anchor links ("a href="#section-id"") that allow users to jump directly to specific sections, improving usability for keyboard and screen reader users. Images have descriptive alt attributes, providing context for users who rely on screen readers or have images disabled. The responsive design, achieved with max-width: 100% and height: auto, ensures the page is accessible on various devices, including mobile and tablets. The "Go to Top" button, styled with position: fixed, provides an easy way for users to return to the top of the page without excessive scrolling. Additionally, the color scheme uses high-contrast colors, such as dark text on a light background, to improve readability for users with visual impairments. These design choices ensure the page is inclusive and accessible to a wide range of users.

Challenges and Lessons Learned

During the implementation of the marine conservation page, I faced several challenges, such as ensuring the sidebar remained fixed without overlapping content and making images responsive across different screen sizes. To resolve the sidebar issue, I used position: fixed and adjusted the margin-left of the main content, which maintained a clean layout. For the images, I applied max-width: 100% and height: auto to ensure they scaled properly without distortion. Another challenge was positioning the "Go to Top" button effectively, which I fixed by using position: fixed and fine-tuning its placement. These challenges taught me the importance of testing on multiple devices and using CSS properties like position and max-width creatively. Overcoming these issues improved my problem-solving skills and deepened my understanding of responsive design, which will be invaluable for future projects.

Compliance

The pages I created aim to comply with Janet regulations by focusing on accessibility, usability, and security. I used semantic HTML tags like "header" and "section" to improve screen reader navigation, added descriptive alt attributes to images, and ensured high-contrast color schemes for better readability. For usability, I implemented responsive design techniques, such as max-width: 100% for images, and tested the pages across devices to ensure functionality. While the pages are static and do not handle sensitive data, I recognize that adding features like forms would require HTTPS and input validation to meet security standards. Although I made significant efforts to comply, I acknowledge areas for improvement, such as adding ARIA labels and conducting user testing for accessibility. This experience taught me the importance of adhering to web standards and the need for continuous improvement to ensure full compliance.
